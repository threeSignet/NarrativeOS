# NarrativeOSPlus 前端设计语言 v1.1

> 定稿日期：2026-05-22
> 状态：终稿确认
> 设计稿：`demo.html`（可交互参考，与本文档同等权威）

---

## 1. 设计定位

**「冰晶之下的精密仪器」** — 磨砂玻璃包裹精密功能模块，边界锐利如切割，内容呼吸其中。

基础但不简陋，前沿但不花哨。所有视觉服务于写作体验。

---

## 2. 核心原则

1. **编辑器是宇宙中心** — 编辑器区域是唯一的舞台，宽度不被任何组件压缩
2. **引擎融合不分离** — 不存在独立的引擎页面，所有引擎数据以浮层/弹窗形式存在
3. **浮层不挤压** — 章纲、设定等组件是 `position: absolute` 浮层，飘在编辑器空白区域，不改变编辑器布局
4. **导航走左侧正文列表** — 没有 editor tab，点哪里看哪里
5. **MOU 是强制弹窗** — 系统级中断，遮罩冻结一切，必须裁决
6. **Companion 是全引擎入口** — 右侧 AI 伙伴带上下文感知，可操控一切
7. **流式响应在当前视图** — 写正文就流在编辑器里，写大纲就流在大纲视图里，不切换视图

## 3. 布局架构

### 3.1 三栏结构

```
┌──────────────────────────────────────────────────────────┐
│  顶栏 (52px): 品牌 | 项目名 ... Token | 模型 | 头像     │
├──────┬───────────────────────────────────┬───────────────┤
│ 图标  │           编辑器（中心）           │  AI 伙伴面板   │
│ 轨道  │  面包屑导航                        │  (360px)      │
│(52px) │  右上角操作按钮(竖排浮层)          │  上下文感知    │
│      │  章纲浮层(左侧abs)                 │  对话式交互    │
│      │  正文(max-780px居中)               │               │
├──────┴───────────────────────────────────┴───────────────┤
│  底部栏 (32px): 心流状态 | Token | 模型 | 提案/伏笔/记忆 │
└──────────────────────────────────────────────────────────┘
```

### 3.2 左侧图标轨道

- 宽度 52px，只显示 Lucide 图标（不用 emoji）
- 三个入口：世界设定(Globe2)、正文(BookOpen)、设置(Settings)
- 左下角心流状态指示点（圆点 + 脉冲动画）
- 点击图标弹出玻璃浮层（从左侧滑出，覆盖编辑器区域）

### 3.3 编辑器区域

- **面包屑导航**代替 tab：`第二卷：深渊 > 第5章 深渊之门` + 状态圆点
- **右上角竖排圆形操作按钮**（abs浮层，不占编辑器空间）
  续写、润色、对话优化、修正、展开、去AI味、查记忆、检查一致性
  hover 显示操作名 tooltip，平板/手机直接 tap
- **章纲浮层**：`position: absolute`，左侧，独立边框圆角，不影响编辑器宽度和布局
- **正文**：`max-width: 780px` 居中，`padding: 40px 64px`，永远保持原有宽度
- 顶栏不显示卷章信息（去重），位置信息唯一由面包屑承载

### 3.4 响应式断点

| 断点 | 左栏 | 编辑器 | Companion |
|------|------|--------|-----------|
| ≥1440px | 52px 图标轨道 | flex-1 | 固定 360px |
| 1024~1439px | 52px | flex-1 | 固定 320px |
| 768~1023px | 48px | flex-1 | 隐藏，底部按钮唤起 |
| <768px | 隐藏，汉堡菜单 | 全屏 | 隐藏，浮动按钮唤起 |

## 4. 视觉规范

### 4.1 色彩体系

```
背景: #0c0c14 (深空黑)
面板: rgba(255,255,255, 0.03)
面板悬停: rgba(255,255,255, 0.06)
面板活跃: rgba(255,255,255, 0.08)

边线: rgba(255,255,255, 0.06) → 0.12 hover → 0.20 active
焦点: rgba(120,160,255, 0.40)

功能色:
  冰蓝 #7dd3fc — 角色、交互高亮、写作中
  薄荷 #86efac — 定稿、批准
  冰红 #fca5a5 — 驳回、风险
  暖黄 #fde68a — 草稿、商讨、伏笔
  淡紫 #c4b5fd — AI 运算中
  暖橙 #fdba74 — 势力

实体标记色:
  角色 → #7dd3fc 底色 + 下划线
  地点 → #86efac
  物品 → #c4b5fd
  势力 → #fdba74
  功法 → #fca5a5
  伏笔 → #fde68a (虚线下划线)

状态圆点（与标签同色）:
  绿=定稿、黄=草稿、蓝=写作中(脉冲)、紫=有章纲、空圈=未创建
  后续可能去掉文字标签，只保留圆点

文字:
  主文字: rgba(255,255,255, 0.88)
  次文字: rgba(255,255,255, 0.50)
  暗淡:   rgba(255,255,255, 0.22)
```

### 4.2 玻璃效果

```css
/* 基础面板 */
background: rgba(255, 255, 255, 0.03);
backdrop-filter: blur(20px) saturate(1.2);
border: 1px solid rgba(255, 255, 255, 0.06);
border-radius: 12px;
box-shadow: 0 4px 16px rgba(0, 0, 0, 0.3);

/* MOU 弹窗（更实心） */
background: rgba(16, 16, 30, 0.96);
backdrop-filter: blur(40px) saturate(1.3);
border-radius: 16px;
box-shadow: 0 24px 80px rgba(0, 0, 0, 0.6);

/* 章纲浮层面板 */
background: rgba(12, 12, 20, 0.85);
backdrop-filter: blur(20px);
border: 1px solid rgba(255, 255, 255, 0.06);
border-radius: 12px;
```

- 玻璃间隙固定 8px
- 不使用发光边框，只用边线对比度
- 悬停只动边线亮度 + translateY(-1px)

### 4.3 字体

- 品牌/大标题: Instrument Serif
- 界面/正文: DM Sans
- 代码/数据: JetBrains Mono
- 中文回退: Noto Serif SC → system-ui

### 4.4 图标

- 全部使用 Lucide 图标库，不使用 emoji

### 4.5 动效

- 快速 150ms，标准 250ms，缓慢 400ms
- 缓动: cubic-bezier(0.4, 0, 0.2, 1)
- 只做有功能含义的动画，不做装饰性动画

## 5. 核心组件

### 5.1 正文列表（左侧浮层）

- 顶部全局入口：设定集、大纲总览
- 按卷分组，卷头显示卷名 + 总字数
- 每卷下有卷纲入口
- 每章右侧显示状态圆点 + 字数
- 状态圆点颜色：绿=定稿、黄=草稿、蓝=写作中、紫=有章纲、空圈=未创建
- 点击导航：设定集→设定视图、大纲→大纲视图、卷纲→卷纲视图、章纲→章纲视图、章节→写作态

### 5.2 编辑器视图（按左侧列表点击切换）

| 视图 | 触发 | 内容 |
|------|------|------|
| 设定集 | 点「设定集」 | 双列卡片网格（世界观、势力、等级、货币、地理、规则） |
| 大纲总览 | 点「大纲总览」 | 卷级概览卡片 + 进度条 + 章节状态标签 |
| 卷纲 | 点某卷「卷纲」 | 按阶段分组，每章标题+摘要+状态 |
| 章纲 | 点某章（有章纲） | 出场角色、场景、关键事件、关联伏笔、世界规则 + 「开始写作」按钮 |
| 写作态 | 章纲→开始写作 | 正文编辑器(原有宽度) + 左侧章纲浮层(abs) + 进度条 |
| 空章节 | 点未写章节 | 空状态页：AI生成章纲 / 手动创建 / 跳过直接写 |

### 5.3 章纲浮层组件（写作态）

```
position: absolute
left: 12px, top: 56px, bottom: 12px
width: 210px
独立边框 + 圆角 + 磨砂背景
overflow-y: auto（固定高度可滚动）
不影响编辑器正文宽度和布局
```

内容：摘要 → 出场角色 → 场景 → 关键事件进度（✓已完成 / ←当前 / 暗灰未到）→ 伏笔 → 约束规则

### 5.4 选中文字工具条

- 上方快捷按钮：续写/润色/修正/展开/去AI味
- 下方输入框 + 发送按钮：自然语言描述需求
- 整体为玻璃浮层，宽 320px

### 5.5 MOU 强制弹窗

- 全屏遮罩 `rgba(0,0,0,0.6) + backdrop-filter: blur(8px)` + 居中弹窗
- 弹窗内容（从上到下）：
  1. **场景上下文**：地理位置（层级链）、故事时间（纪年/时节/距开篇）、相关角色（在场/附近/未知）
  2. **AI 理由**
  3. **提案内容**（正文预览 / diff）
  4. **关联信息**：引用实体、关联伏笔、Token消耗、置信度、谏官备注、应用规则
  5. **记忆快照**：本章已确认事件
  6. **批示输入框**（textarea）
  7. **三个操作**：驳回 / 商讨 / 批准

- 每个操作都可附带输入内容：
  - 批准+输入 → 通过并微调
  - 驳回+输入 → 拒绝并反馈原因
  - 商讨+输入（必填）→ 不通过不拒绝，AI重新提案

### 5.6 AI 伙伴面板（Companion）

- 顶栏显示上下文标签（当前章节 + 视图 + 涉及实体）
- 纯对话式交互，不承载章纲等结构化信息
- 输入框下方无快捷按钮（快捷指令已移至编辑器右上角）

### 5.7 心流状态

- 左下角图标轨道：小圆点指示器（可点击切换状态 demo）
- 底部栏左侧同步显示状态文字
- 颜色：灰=IDLE、紫脉冲=AI_PROCESSING、黄闪烁=REVIEWING、绿脉冲=EXECUTING

### 5.8 底部栏

- 左侧：心流状态（可点击）+ Token + 模型
- 中间：可点击的功能入口（提案数 / 伏笔数 / 记忆数 / 风险数）
- 右侧：连接状态 + 最后保存时间

## 6. 流式响应规范

### 6.1 核心原则

**流式响应在当前视图内，不切换视图。**

| 场景 | 流式输出位置 |
|------|-------------|
| AI 生成正文 | 直接在编辑器正文区域追加，带紫色左边框 + 闪烁光标 |
| AI 生成章纲 | 在章纲视图内流式输出 |
| AI 生成大纲 | 在大纲视图内流式输出 |
| AI 生成设定 | 在设定视图内流式输出 |

### 6.2 流式视觉效果

- 已完成段落：正常显示
- 正在生成的段落：`border-left: 2px solid var(--accent-violet)` + `opacity: 0.65` + 闪烁光标
- 生成完毕后：左边框消失，透明度恢复正常

### 6.3 滚动行为

- 每个内容区域有固定高度（`overflow-y: auto`），不撑大页面
- 流式生成时自动滚动到底部
